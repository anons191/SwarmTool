## Contract Details

### HTML IDs vs CSS Classes - CRITICAL DISTINCTION

- **html_ids** = UNIQUE elements (one per page) - use `getElementById()` or `querySelector('#id')`
- **css_classes** = REPEATED elements (multiple instances) - use RELATIVE selectors

Example for modals:
```javascript
// Get the UNIQUE modal container by ID
this.modal = document.getElementById('confirm-modal');

// Get elements INSIDE that modal by CLASS (relative lookup)
this.confirmBtn = this.modal.querySelector('.modal-confirm-btn');
this.cancelBtn = this.modal.querySelector('.modal-cancel-btn');
```

This ensures each modal instance finds ITS OWN buttons, not another modal's buttons.

### JavaScript Function Signatures

If js_functions is defined, you MUST:
- DEFINE functions with the exact signature shown
- CALL functions with the exact parameters shown

Example: `createBoard: "(name: string) => Promise<Board>"` means:
- Takes ONE string parameter, NOT an object
- Correct: `createBoard(boardName)`
- WRONG: `createBoard({ name: boardName })`

```javascript

    this.setLoading(true);
    this.data = await fetchData();
    this.render();

    showToast('Failed to load data', 'error');

  }
}
```
